# Local Pilot

The Local Pilot (`scripts/pilot.py`) scans a local input directory for markdown files, text files, and JSONL records, converting them into memory atoms and open loops. It then runs the AhaOS validation, incubation, verification, and report rendering pipeline completely offline without external APIs or LLM dependencies.

## Options

- `--input-dir`: Path to the directory containing raw input files (e.g., markdown, text) to scan.
- `--work-dir`: Path where temporary memory database files, JSONL outputs, and execution states are written. Defaults to `/tmp/ahaos-local-pilot`.
- `--report-dir`: Path where the final insight markdown report is written. Defaults to `/tmp/ahaos-local-pilot-reports`.
- `--top-k`: Maximum number of verified insights to display in the generated report. Defaults to `3`.
- `--dry-run`: Prints the generated Markdown report directly to stdout instead of writing it to `--report-dir`.

## Generated Files

All intermediate outputs are written to `--work-dir`:

- `atoms.jsonl`: Extracted active memory atoms.
- `open_loops.jsonl`: Extracted open loops.
- `state.json`: Execution metadata recording seen atom/loop IDs, `created_at`, `last_seen_at`, `last_triggered_at`, and `activation_count` to support incremental triggers and Ebbinghaus forgetting curve decay.
- `aha_candidates.jsonl`: Weak latent insight candidates generated by association heuristics (`repeated_tag`, `structural_echo`, `open_loop_bridge`).
- `latent_links.jsonl`: Lower-confidence potential associations and connection rationale.
- `trigger_candidates.jsonl`: Insight candidates specifically triggered by newly added memory atoms in the current execution cycle.

## Safety Boundaries & Forbidden Inputs

To preserve safety:
- **Refused directories**: The pilot refuses to scan `$HOME`.
- **Ignored file patterns**: Skips hidden files, hidden directories (e.g., `.git`, `.ssh`), and credential-related paths (e.g., `.env`, `.pem`, `.key`, `token`, `secret`, `password`, `auth`).
- **Content-level secret filtering**: Skips any line containing api key indicators like `sk-`, `ghp_`, `github_pat_`, `AKIA`, `xoxb-`, `BEGIN PRIVATE KEY`, `client_secret`, `refresh_token`, and `access_token` to prevent sensitive credentials from entering the memory database.

## How to Inspect Output

You can inspect the generated files under `--work-dir`:
- Read `trigger_candidates.jsonl` to see what new inputs activated.
- Read `aha_candidates.jsonl` to audit all generated candidates.
- View the generated markdown report under `--report-dir` for verified, high-leverage insights.

## Rollback

To clean up all temporary files and reports generated by the pilot, run:

```bash
rm -rf /tmp/ahaos-local-pilot /tmp/ahaos-local-pilot-reports
```
