# v0.4.0 — Local-first insight engine candidate

This release note is prepared for a future `v0.4.0` release candidate of `AhaOS`.

## Version timeline

This repository already has earlier public version history. The current project metadata and README describe the active scope as `0.4.0` / `v0.4`.

Therefore:

- do not create or publish `v0.1.0` from the current repository state;
- keep existing historical tags intact;
- use this file only when preparing a validated `v0.4.0` tag after local review.

## Candidate release scope

Candidate baseline surfaces:

- local-first insight engine README;
- CLI and local pilot documentation;
- agent checkpoint loop documentation;
- safety model and contribution boundaries;
- pull request template and CODEOWNERS;
- existing examples and skill package surfaces;
- evidence-before-insight contributor boundary.

## Validation before publishing

Run from the repository root when the local environment is available:

```bash
.venv/bin/python -m pytest tests/
.venv/bin/ruff check .
python3 scripts/pilot.py --input-dir examples/local_pilot_input --work-dir /tmp/ahaos-local-pilot --report-dir /tmp/ahaos-local-pilot-reports --top-k 3 --dry-run
git diff --check
git status --short
```

Do not claim these commands passed unless exact output is captured.

## Publishing path

After local `master` is synced to the validated remote HEAD and validation is green:

```bash
git tag -a v0.4.0 -m "v0.4.0 — Local-first insight engine candidate"
git push origin v0.4.0
```

Manual release fallback:

```bash
gh release create v0.4.0 \
  --repo EthanSangSSS/AhaOS \
  --title "v0.4.0 — Local-first insight engine candidate" \
  --notes-file docs/releases/v0.4.0.md \
  --verify-tag
```

## Honesty boundary

This release candidate does not claim hosted memory service behavior, autonomous action execution, production adoption, external users, or cognitive reproduction. It is a local-first candidate for evidence-backed insight generation.

## Known limitations

- Insights are hypotheses until validated against primary evidence.
- Local pilot behavior depends on explicit user-provided directories.
- AhaOS must not scan unrelated home directories, token stores, browser data, or private chat logs.
- Do not publish this release until the local environment and CI evidence are current.
