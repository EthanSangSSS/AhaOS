# v0.1.0 — Local-first insight engine baseline

This release note is prepared for the first public baseline release of `AhaOS`.

## Release scope

Included baseline surfaces:

- local-first insight engine README;
- CLI and local pilot documentation;
- agent checkpoint loop documentation;
- safety model and contribution boundaries;
- pull request template;
- existing examples and skill package surfaces.

## Validation before publishing

Run from the repository root when the local environment is available:

```bash
.venv/bin/python -m pytest tests/
.venv/bin/ruff check .
python3 scripts/pilot.py --input-dir examples/local_pilot_input --work-dir /tmp/ahaos-local-pilot --report-dir /tmp/ahaos-local-pilot-reports --top-k 3 --dry-run
git diff --check
git status --short
```

Do not claim these commands passed unless exact output is captured.

## Publishing path

After local `master` is synced to the validated remote HEAD:

```bash
git tag -a v0.1.0 -m "v0.1.0 — Local-first insight engine baseline"
git push origin v0.1.0
```

Manual release fallback:

```bash
gh release create v0.1.0 \
  --repo EthanSangSSS/AhaOS \
  --title "v0.1.0 — Local-first insight engine baseline" \
  --notes-file docs/releases/v0.1.0.md \
  --verify-tag
```

## Honesty boundary

This release does not claim hosted memory service behavior, autonomous action execution, production adoption, external users, or cognitive reproduction. It is a local-first baseline for evidence-backed insight generation.

## Known limitations

- Insights are hypotheses until validated against primary evidence.
- Local pilot behavior depends on explicit user-provided directories.
- AhaOS must not scan unrelated home directories, token stores, browser data, or private chat logs.
